  • ✗ لا رسوم بيانية أو Charts حية
  • ✗ لا تنبيهات وإشعارات للأدمن

  • ✗ سجل المزايدات وهمي تماماً

❌ ما هو مفقود بالكامل — الثغرات الحرجة

الوحدات الأساسية التي يجب بناؤها لتصبح المنصة منصة مزادات حقيقية

⚠️ تحذير هام

المشروع حالياً عبارة عن نظام إدارة مستخدمين ومحافظ مالية فقط. الجزء الأساسي من أي منصة مزادات — وهو السيارات والمزادات والمزايدة — غائب بالكامل. لا يمكن إطلاق المنصة بدون هذه الوحدات.

🚗 وحدة إدارة المركبات / السيارات

🔴 حرجة — أولوية قصوى

المنصة لا تملك أي جدول أو نموذج أو واجهة للسيارات. لا يمكن عرض أي سيارة للمزايدة.

  • جدول vehicles: الماركة، الموديل، السنة، اللون، الكيلومترات، رقم الهيكل (VIN)
  • رفع صور متعددة للسيارة مع الصورة الرئيسية
  • بيانات الحالة: جديد، مستعمل، تالف، بحاجة إصلاح
  • تقرير الفحص الفني وشهادات الملكية
  • نظام تصنيف: ماركة، فئة، سنة، السعر الابتدائي
  • واجهة الأدمن لإضافة وإدارة السيارات

🔨 محرك المزادات الأساسي

🔴 حرجة — أولوية قصوى

لا يوجد أي جدول auctions أو bids في قاعدة البيانات. المنصة لا تستطيع إجراء أي مزاد.

  • جدول auctions: وقت البدء والانتهاء، السعر الابتدائي، أدنى زيادة، سعر التحفظ
  • ربط المزاد بالسيارة وبالمزاد المنظِّم (بائع/منصة)
  • حالات المزاد: scheduled / live / ended / cancelled
  • جدول bids: المزايد، المبلغ، الوقت، الحالة (فائز/عادي)
  • منطق تحديد الفائز تلقائياً عند انتهاء المزاد
  • نظام التأمين (Deposit) المطلوب للدخول في المزاد

📡 المزايدة اللحظية Real-Time Bidding

🔴 حرجة — أولوية قصوى

أي منصة مزادات تتطلب تحديثات فورية. المزايد يجب أن يرى السعر يتغير فوراً دون تحديث الصفحة.

  • إعداد Laravel Broadcasting مع Pusher أو Laravel WebSockets
  • قناة BidPlaced تُبث لجميع المشاركين في المزاد
  • عداد تنازلي حي متزامن مع الخادم
  • تنبيه "لقد تجاوزك شخص آخر" لحظياً
  • تحديث عدد المزايدين وآخر سعر دون reload

🔔 نظام الإشعارات

🟡 عالية الأولوية

لا يوجد نظام إشعارات داخل التطبيق. المستخدم لا يُبلَّغ بأي أحداث.

  • إشعارات داخل التطبيق (Database Notifications)
  • إيميل عند الفوز بالمزاد أو التجاوز
  • إشعار انتهاء وقت المزاد قرب الانتهاء
  • تنبيه الأدمن بطلبات الإيداع الجديدة
  • إشعار الفائز بإتمام عملية الدفع والاستلام

💳 بوابات الدفع المتكاملة

🟡 عالية الأولوية

المنصة تعتمد على التحويل البنكي اليدوي فقط. لا دفع إلكتروني فوري.

  • دمج Moyasar أو HyperPay لدفع التأمين المطلوب للمزاد
  • دفع قيمة السيارة الفائزة إلكترونياً
  • استرداد التأمين للخاسرين تلقائياً
  • سجل مفصل للمدفوعات وإيصالات رقمية

🌐 صفحات استعراض المزادات للعامة والمزايدين

🟡 عالية الأولوية

المزايد ليس لديه أي صفحة لاستعراض المزادات الحالية والمشاركة فيها.

  • صفحة قائمة المزادات مع فلترة (ماركة، حالة، سعر)
  • صفحة تفاصيل المزاد مع معرض الصور والمواصفات
  • زر "زايد الآن" مع خانة إدخال السعر
  • سجل المزايدات الحية داخل صفحة المزاد
  • قائمة مزاداتي والفوز والمتابعة

📊 تقارير وإحصائيات متقدمة

🔵 متوسطة الأولوية

لوحة تحكم الأدمن تعرض أرقاماً بسيطة فقط بدون رسوم بيانية أو تقارير.

  • Chart.js: إيرادات يومية/أسبوعية/شهرية
  • عدد المزادات المنتهية والنشطة والمجدولة
  • أعلى المزايدين نشاطاً وأكثر السيارات طلباً
  • تصدير التقارير إلى Excel و PDF

🔑 ميزات إضافية لمنصة احترافية

🔵 متوسطة الأولوية

ميزات تميّز المنصة الاحترافية عن الأنظمة الأساسية.

  • قائمة المراقبة (Watchlist) لحفظ المزادات
  • نظام التقييم بعد إتمام الصفقة
  • بائع الطرف الثالث (Seller account)
  • نظام العمولة وتوزيع الأرباح
  • تطبيق جوال أو PWA
  • خريطة المزادات حسب الموقع الجغرافي
